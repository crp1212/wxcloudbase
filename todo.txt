数据交互模式
第一步、
1. 客户端进入，携带信息： 身份、房间号
2. 服务端接收，确认房间、发送房间历史数据
3. 客户端接收历史数据，渲染当前实时态

第二步、
1. 客户端发送操作，服务器接收，记录当前操作数据，广播给房间内其他人
2. 客户端接收到信息，显示接收到的信息

消息数据结构
{
  type: 'init', // 初始化
  guid: '', // 用户唯一id
}

{
  type: 'history', // 历史信息
  data: [] // 历史信息数组
}

{
  type: 'step', // 步骤，代表已每一步操作
  from: 'user', // 系统或者用户
  data: 0, // 翻开索引
}
